Help

HTTP Proxy Settings:

  • If your server cannot access the Sora API directly, configure an HTTP proxy.
  • Proxy authentication is optional. Leave blank if your proxy does not require it.
  • In Docker, the proxy host is often host.docker.internal instead of 127.0.0.1.

About Image Localization:

  • This feature solves issues where Sora image URLs are inaccessible by saving generated images to the local server.
  • When enabled, images returned by Sora are automatically downloaded to the specified directory.
  • API image links will be replaced with local URLs.
  • Ensure the save directory has sufficient disk space and permissions.

Note: Configuration changes take effect immediately but do not affect images already generated.
